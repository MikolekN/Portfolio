Projects

  • User Behviour Driven Interface

    Master's degree project developed by a two-person team, focused on implementing a functional model of a banking application that continuously learns individual user workflows and dynamically adapts its UI to suggest the next likely action.
    We built a responsive front end in TypeScript and React, leveraging a component-based architecture to render and update interface elements in real time as user behavior patterns evolve.
    User interactions — including clicks, hovers, and page transitions — are captured via JavaScript event listeners (e.g., click, mouseover) and streamed to the backend for logging.
    On the server side, a Flask application in Python gathers user interactions, persists them, analyzes them using process mining techniques from the PM4Py library, and exposes RESTful endpoints for analytics.
    We use PM4Py to extract XES-formatted event logs from a MongoDB datastore and apply process-mining algorithms to discover and conformance-check personalized user process models.
    A Camunda BPMN engine then consumes these models to predict the most probable next steps and to flag when users search for actions in unexpected interface locations.
    Finally, the entire application stack is containerized using Docker to ensure consistent deployment across development, testing, and production environments.

  • Process Mining Chrome Extension

    Bachelor’s degree project developed in a two-person team, focused on creating a Chrome browser extension that analyzes user browsing history and transforms it into visual workflow models using process mining techniques.
    The front end, implemented as a Chrome extension, collects browsing history via the browser’s API, offering users filtering options (such as selecting a specific date range) before sending the refined data to the backend.
    The extension also allows users to view their browsing history, access usage statistics (e.g., frequency of website visits), download raw history data or event logs in XES format, and most importantly, explore a visualized process model showing how they navigate the web.
    The backend service, built to complement the extension, processes the received data into an event log, converts it into the XES standard format, and leverages the PM4Py library to discover and visualize the user’s process model.
    Generated visualizations and event logs are then made available for download or direct viewing through the extension, providing users with insightful, structured representations of their browsing behavior.

  • Deep Learning Basics

    Project developed in a two-person team, aimed at designing and training a deep learning model capable of classifying Polish road signs based on images.
    We built and trained the model using the Keras API along with libraries like NumPy and Matplotlib, while managing the project codebase through GitHub.
    Throughout development, we leveraged Weights & Biases (wandb) for experiment tracking, model checkpointing (saving intermediate models after each epoch), and hyperparameter optimization using wandb sweeps.
    Our experiments focused heavily on fine-tuning the model by configuring and running sweeps to explore different hyperparameter settings.
    The model was trained on two datasets sourced from Kaggle and validated using a manually collected test set of 687 photographs.
    Performance evaluation included key metrics such as accuracy, precision, recall, and F1-score, both overall and broken down per class, providing a detailed view of the model's classification capabilities.

  • PG IUI NER API

    Project developed in a team of 5 people, aimed at building an intelligent text-processing service for Polish and English. The service implements Named Entity Recognition (NER) functionality, allowing both entity extraction (e.g., people, organizations, locations) and custom model training using user-provided datasets.
    Built on top of a FastAPI REST framework in Python, the service integrates deep learning models based on Transformer architectures (RoBERTa) and uses libraries like Hugging Face Transformers, PyTorch, Datasets, and APScheduler for model management and job scheduling.
    We implemented two base models—one for Polish and one for English—and allowed users to fine-tune models on new data through a scalable training queue, monitored and managed by cron jobs. Training details, such as current epoch and total number of epochs, are accessible via the API.
    Models were trained on datasets CoNLL-2003 and KPWR-NER, with training conducted using Hugging Face’s roberta-large and polish-roberta-large-v2 models.
